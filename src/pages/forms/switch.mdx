# Switch

A switch allows users to turn a setting on or off.

A switch is similar to a checkbox, but communicates a binary state such as:

- Enabled / Disabled
- On / Off
- Active / Inactive

Use a switch when the action immediately affects a setting.

## Accessibility Notes

- Use a visible label.
- Use `role="switch"` when representing an on/off setting.
- Keep the checked state synchronized with the UI.
- Prefer native form controls over custom implementations.

---

## Basic Switch

```html
<label>
  <input type="checkbox" role="switch" />

  Email Notifications
</label>
```

Screen reader:

```txt
Email Notifications
switch
off
```

When checked:

```txt
Email Notifications
switch
on
```

## Switch vs Checkbox

These controls are visually similar but communicate different intent.

### Checkbox

```html
<input type="checkbox" />
```

Screen reader:

```txt
checkbox
checked
```

Use for:

- Selecting options
- Multi-select forms
- Preferences

Examples:

```txt
☑ Email
☑ SMS
☐ Push Notifications
```

---

### Switch

```html
<input type="checkbox" role="switch" />
```

Screen reader:

```txt
switch
on
```

Use for:

- Feature toggles
- Settings
- Enable / Disable actions

Examples:

```txt
Dark Mode
On

Email Notifications
Off
```

## Keyboard Support

Native switches inherit keyboard support from checkboxes.

| Key         | Action                   |
| ----------- | ------------------------ |
| Tab         | Focus switch             |
| Shift + Tab | Move to previous element |
| Space       | Toggle switch            |

## Common Mistakes

### ❌ Clickable Div

```html
<div class="switch" onclick="toggle()"></div>
```

Problems:

- Not keyboard accessible.
- No semantic meaning.
- Screen readers do not recognize it as a switch.

---

### ✅ Native Control

```html
<input type="checkbox" role="switch" />
```

Provides:

- Keyboard support
- Screen reader support
- Form integration

---

### ❌ Missing Label

```html
<input type="checkbox" role="switch" />
```

Screen readers may announce:

```txt
switch
off
```

without explaining what the switch controls.

---

### ✅ Visible Label

```html
<label>
  <input type="checkbox" role="switch" />

  Email Notifications
</label>
```

Screen reader:

```txt
Email Notifications
switch
off
```

---

### ❌ State Not Updated

```html
<input type="checkbox" role="switch" aria-checked="true" />
```

while the visual UI shows OFF.

Assistive technologies and visual users receive conflicting information.

---

### ✅ Keep State Synchronized

```html
<input type="checkbox" role="switch" checked />
```

The visual and accessible states should always match.

---

## When to Use

Use a switch when users are toggling a setting between two states.

Examples:

- Dark Mode
- Email Notifications
- Auto Save
- Two-Factor Authentication
- Location Services

Prefer a checkbox when users are selecting options in a form.

Think of it as:

```txt
Checkbox = Select this option

Switch = Turn this setting on or off
```
