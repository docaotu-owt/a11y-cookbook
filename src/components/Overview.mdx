# Accessibility Cookbook

## What is Accessibility?

Web accessibility (often abbreviated as **a11y**) is the practice of designing and developing websites that can be used by everyone, including people with disabilities.

Accessible applications support users with various needs, including:

* Screen reader users
* Keyboard-only users
* Users with low vision
* Users with color vision deficiencies
* Users with motor impairments
* Users with cognitive impairments

Accessibility is not a separate feature. It is a fundamental aspect of good user experience and software quality.

---

## About This Cookbook

Accessibility Cookbook is a practical reference guide for implementing accessibility in modern frontend applications.

Rather than focusing solely on theory, this cookbook provides interactive examples and real-world implementation patterns that developers can apply directly in their projects.

Each topic includes:

* Interactive examples
* Source code samples
* Good vs Bad implementations
* Accessibility notes and best practices
* Keyboard support documentation
* Screen reader behavior examples

## Topics

### HTML Foundations

Use semantic HTML whenever possible. Native HTML elements provide built-in accessibility and should always be preferred over custom implementations.

Topics include:

* Button
* Link
* Heading
* Image
* Table
* List

---

### Forms

Forms are one of the most common areas where accessibility issues occur.

Topics include:

* Label
* Input
* Select
* Textarea
* Checkbox
* Radio Group
* Switch
* Error Messages
* Fieldset / Legend

---

### ARIA

ARIA (Accessible Rich Internet Applications) provides additional semantic information when native HTML alone is insufficient.

Topics include:

* aria-label
* aria-labelledby
* aria-describedby
* aria-expanded
* aria-current
* aria-live

> Use ARIA only when native HTML cannot provide the required semantics.

---

### Components

Modern web applications often contain complex interactive components that require additional accessibility considerations.

Topics include:

* Modal
* Accordion
* Tabs
* Dropdown Menu
* Tooltip
* Combobox
* Toast
* Data Table

---

## Accessibility Principles

This cookbook follows several core accessibility principles:

1. Prefer semantic HTML over custom elements.
2. Ensure all functionality is keyboard accessible.
3. Provide meaningful labels and descriptions.
4. Maintain a logical document structure.
5. Manage focus appropriately.
6. Support assistive technologies such as screen readers.
7. Use ARIA only when necessary.

---

## How to Use This Cookbook

For each topic:

1. Review the example implementation.
2. Compare good and bad patterns.
3. Read the accessibility notes.
4. Test keyboard interactions.
5. Test with a screen reader such as VoiceOver or NVDA.

Understanding accessibility requires both implementation knowledge and hands-on testing. This cookbook is intended to help developers build that practical experience.
